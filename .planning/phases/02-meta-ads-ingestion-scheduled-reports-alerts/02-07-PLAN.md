| Boundary | Description |
|----------|-------------|
| Telegram user → /report handler | User is pre-validated by AllowlistMiddleware before handler body executes |
| scheduler job registration → SQLAlchemyJobStore | Job functions are module-level callables; no non-serializable args stored |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-22 | Elevation of Privilege | /report handler | mitigate | AllowlistMiddleware is registered BEFORE the router in setup.py; every /report invocation is blocked for non-allowlisted senders before the handler body executes — no additional check needed in handler |
| T-02-23 | Tampering | APScheduler job registration | mitigate | Job functions are module-level callables (meta_ingest_module.meta_ingest_job etc.) — serializable by APScheduler; no bot/db objects in args; register_job_resources called synchronously before scheduler.start() |
| T-02-24 | Denial of Service | /report race with scheduled job | accept | max_instances=1 on all 3 scheduler jobs prevents concurrent APScheduler execution; /report manual trigger calls _run_daily_report directly (not via scheduler) so APScheduler instance limit does not apply — acceptable for v1 single-user deployment |
</threat_model>
